# Safo Restaurant Landing Site

Сайт-визитка для горного ресторана **Safo** при резорте *Zarra Resort* (Самарканд). 
Проект полностью оптимизирован под мобильные устройства, содержит мультиязычное интерактивное меню (RU, UZ, EN), умный статус рабочего времени, живой поиск по блюдам, галерею картинок и интерактивный конструктор заказа (корзину).

---

## 🚀 Как выгрузить сайт на GitHub и запустить на Netlify

Вы можете развернуть этот сайт в сети бесплатно за пару минут, используя связку **GitHub** + **Netlify**. При любом изменении файлов на вашем компьютере и отправке их в GitHub, Netlify будет автоматически обновлять сайт.

### Шаг 1: Создание репозитория на GitHub
1. Перейдите на [github.com](https://github.com/) и войдите в свой аккаунт (или зарегистрируйтесь).
2. Нажмите кнопку **New** (Создать новый репозиторий).
3. Укажите имя репозитория (например, `safo-restaurant`).
4. Оставьте его **Public** (публичным) или сделайте **Private** (приватным).
5. Не добавляйте файлы README, .gitignore или лицензию (они уже настроены либо не требуются). Нажмите **Create repository**.

### Шаг 2: Инициализация Git и отправка кода
Откройте ваш терминал на компьютере в папке проекта `safo` и выполните следующие команды:

```bash
# 1. Инициализируем локальный репозиторий Git
git init

# 2. Добавляем все файлы проекта под версионный контроль
git add .

# 3. Делаем первый коммит изменений
git commit -m "Initial commit: premium version with search, cart and localization"

# 4. Переименовываем ветку по умолчанию в main
git branch -M main

# 5. Связываем локальный репозиторий с вашим репозиторием на GitHub
# (ЗАМЕНИТЕ ссылку ниже на ту, которую выдал вам GitHub после создания репозитория)
git remote add origin https://github.com/ВАШ-ЛОГИН/safo-restaurant.git

# 6. Отправляем изменения на GitHub
git push -u origin main
```

---

### Шаг 3: Деплой на Netlify
1. Перейдите на сайт [netlify.com](https://www.netlify.com/) и войдите под вашим аккаунтом GitHub (это самый простой способ).
2. В панели управления Netlify нажмите кнопку **Add new site** -> **Import an existing project**.
3. Выберите провайдера **GitHub** (понадобится дать Netlify разрешение на доступ к репозиториям).
4. Выберите из списка ваш репозиторий `safo-restaurant`.
5. Настройки сборки оставьте пустыми (так как это чистый статический HTML/CSS/JS сайт без фреймворков):
   * **Build command:** (пусто)
   * **Publish directory:** `.` (или оставьте по умолчанию пустое поле)
6. Нажмите кнопку **Deploy safo-restaurant**.
7. Через 10-15 секунд ваш сайт будет опубликован! Netlify выдаст вам случайный публичный адрес (например, `https://fancy-gelato-12345.netlify.app`).

### Шаг 4: Настройка красивого домена на Netlify
Если вы хотите переименовать выданный Netlify домен на более красивый (например, `safo-restaurant.netlify.app`):
1. В панели управления сайтом на Netlify перейдите в раздел **Site configuration** -> **Domain management**.
2. В блоке *Custom domains* нажмите на троеточие рядом со стандартным доменом Netlify и выберите **Edit site name**.
3. Введите желаемое имя (например, `safo-restaurant`) и нажмите **Save**.

## 🛠 Технические особенности проекта
* **Zero-dependency:** Сайт написан на чистом HTML5, CSS3 и Vanilla JS. Он не зависит от внешних библиотек и загружается мгновенно на мобильных устройствах.
* **Локальные шрифты:** Семейства шрифтов `Manrope` и `Playfair Display` хранятся локально в `assets/fonts`, чтобы сайт открывался быстро независимо от доступности CDN Google Fonts.
* **Адаптивность:** Mobile-first адаптив (`max-width: 480px`) со скругленным контуром на планшетах и десктопах, имитирующий приложение.
* **Умный статус:** Статус работы ресторана (открыто/закрыто) вычисляется на лету на основе времени на устройстве гостя (рабочие часы: с 11:00 до 23:00).
* **Синхронизация языка:** Выбор языка (RU, UZ, EN) автоматически сохраняется в `localStorage` браузера и применяется ко всем страницам.
